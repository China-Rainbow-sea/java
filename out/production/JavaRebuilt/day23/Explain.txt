

集合:
 什么是集合，有什么用 ？？？
 数组其实就是一个集合，集合实际上就是一个容器，可以用来容纳其他类型的数据。

 集合为什么说在开发中使用较多。
  集合是一个容器，是一个载体，可以一次容纳多个对象，
  在实际开发中，假设连接数据库，数据库当中有10条记录，那么假设把这10条记录查询出来，在Java程序中会将10条，
  数据封装成10个记录查询出来，在java程序中会将10条，数据封装成10个Java对象，然后将10个Java程序中会将10条
  数据封装成10个对象，然后将10个Java对象放到某一个集合当中，将集合传到前端，然后遍历集合，将一个数据一个
  数据展现出来。


  1.2 集合不能直接存储基本数据类型，另外集合也不能直接存储Java对象，集合当中存储的都是Java对象的内存地址，
  (或者说集合中存储的引用的list.add(100)) // 自动装箱 Integer
  注意:
      集合在Java中本身是一个容器，是一个对象，集合中任何时候存储的都是引用。
      集合中任何时候存储的都是“引用”对象的地址。

  2.3 在Java中每一个不同的集合，底层会对应不同的数据结构当中。什么是数据结构？数据存储的结构就是数据结构
  不同的数据结构，数据存储方式不同，例如：
  数组，二叉树，链表，哈希表，以上这些都是常见的数据结构。
  你往集合c1中放数据，可能是放到数组上了。
  你往集合c2中放数据，可能是放到二叉树上了。
  你使用不同的集合等同于使用了不同的数据结构。Java中有已经将数据结构实现了，
  已经写好了，这些常用的集合类，你只需要掌握怎么用？在什么情况下选择哪一种合适的集合去使用即可

  new ArrayList :创建一个集合，底层数组，
  new LinkedList(): 创建一个集合对象，底层是链表。
  new TreeSet(): 创建一个集合对象，底层是二叉树.

  1.4 集合在java JDK 中哪个包下？？
    java.util.*;
    所有的集合类和集合接口都在 java.util.*包下，
  1.5 为了让大家掌握集合这块的内容，最好能将集合的继承结构图背会。

  1.6 在Java中集合分为两大类:
    1. 一类是单个方式存储元素。
       单个方式存储元素，这一类集合中超级父类接口: java.util.Collection;

    2. 一类是以键值对儿的方式存储元素，这一类集合中超级父接口: java.util.Map;

    总结(所有的实现类)
      ArrayList 底层是数组,
      LinkedList 底层是双向链表
      Vector: 底层是数组，线程安全的。
      HashSet：底层是HashMap 放到HashSet 集合中的元素等同于放到 HashMap 集合key 部分了.
      TreeSet: 底层 TreeMap,放到 TreeSet 集合中的元素等同于放到 TreeMap 集合key 部分了.

      HashMap 底层是哈希表
      Hashtable 底层也是哈希表，只不过线程安全的，效率较低，使用较小。
      Properties: 是线程安全的，并且key 和 value 只能存储字符串 String,
      TreeMap: 底层是二叉树，TreeMap 集合的key 可以自动按照大小顺序排序。


      List 集合存储元素的特点：
       有序可重复
       有序，存进入的顺序和取出的顺序相同（每个数据的存储都是连续有序的），不是
       可重复: 存进去了 1，可以再存储一个 1

       Set 集合存储元素的特点:
          无序不可重复的
          无序，每个数据的存储不是连续的，而是散列的，另外 Set 集合中元素没有下标，
          不可重复，存进去 1，不能再存储1了...
       SortedSet 存储元素的特点
         首先是无序不可重复的，但是SortedSet 集合中的元素是可排序的。
         无序，存进去的顺序和取出的顺序不一定相同，另外 Set 集合中元素没有下标，
         不可重复，存进去1，不能再存储1了。
         可排序，可以按照大小顺序排列。

       Map 集合的key,就是一个Set 集合，
       注: Set 集合中放数据，实际上放到了 Map 集合的 Key 部分...
